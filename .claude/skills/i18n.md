# i18n Module Skill

This skill helps you work with internationalization in OpenixWidgets.

## i18n System Overview

### Supported Languages
- zh-CN (Simplified Chinese) - **Default/fallback language**
- zh-TW (Traditional Chinese)
- en-US (English)
- ja-JP (Japanese)
- ko-KR (Korean)

### Configuration
- Location: `src/i18n/index.ts`
- Uses i18next with react-i18next
- Auto-detects browser language
- Stores preference in localStorage

### Translation Files
Location: `src/i18n/locales/`
- `zh-CN.json` - Primary language (keys return themselves if missing)
- `zh-TW.json`, `en-US.json`, `ja-JP.json`, `ko-KR.json`

## i18next-cli Commands

```bash
# Extract translation keys from code
npm run i18n:extract

# Check translation status and coverage
npm run i18n:status

# Lint for hardcoded strings
npm run i18n:lint

# Generate TypeScript types
npm run i18n:types
```

## Configuration File

`i18next.config.ts`:
- Defines supported locales
- Configures extraction from `src/**/*.{js,jsx,ts,tsx}`
- Outputs to `src/i18n/locales/{{language}}.json`
- Sets zh-CN as default value source

## Common Tasks

### Using translations in components
```tsx
import { useTranslation } from 'react-i18next';

const MyComponent = () => {
  const { t, i18n } = useTranslation();

  return (
    <div>
      <h1>{t('settings.title')}</h1>
      <button onClick={() => i18n.changeLanguage('en-US')}>
        English
      </button>
    </div>
  );
};
```

### Adding new translation keys
1. Add key to `src/i18n/locales/zh-CN.json` with Chinese text
2. Run `npm run i18n:extract` to propagate to other languages
3. Translate the empty strings in other language files
4. Run `npm run i18n:status` to check coverage

### Changing language
```tsx
import { useTranslation } from 'react-i18next';

const { i18n } = useTranslation();
i18n.changeLanguage('en-US');
```

## Important Notes

- Always use `t()` function for user-facing text
- Never hardcode strings that should be translated
- Run `npm run i18n:lint` to find hardcoded strings
- zh-CN is the source language - other languages translate from it
- Translation keys use dot notation: `settings.title`, `sidebar.expand`
